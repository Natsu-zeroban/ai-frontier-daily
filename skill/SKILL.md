# frontier-daily · AI 前沿日报维护

把"文章链接 → 中文摘要 → 更新网站 → 部署上线"这条流水线固化下来，保证每次风格一致、
结构不坏、发布可控。**风格是这个 skill 的核心资产**，写摘要前必读 `references/STYLE.md`。

## 这个网站是什么
- 单文件静态站 `index.html`(仓库根)，日报式：多期(VOL.01/02/03…)可在同一页切换。
- 每期含若干"精读卡片"(干货/观点，默认折叠只露标题+核心)，可选"要闻茶点"(要闻一句话)。
- JS 数据驱动：期数/篇数/数字键/命令行显示全部从 DOM 自动生成，加新期只改两处 HTML。
- 部署在 GitHub Pages，`git push` 后约 1 分钟自动更新。

## 前置：先读这两份
- `references/STYLE.md` —— 写作与呈现铁律(分类型、大白话、翻黑话、层级、只收指定链接)。**每次写摘要前读。**
- `references/TEMPLATE.md` —— 加文章/加茶点/新开一期的精确 HTML 片段 + 发布 checklist。**动手改 HTML 前读。**

## 工作流(五步)

### 1. 定位与自检
- 确认在本站仓库目录(含 `index.html` 和 `skill/`)。首次或换机器时跑 `python skill/scripts/check_env.py`
  确认 python/curl/git 就绪、origin 指向用户自己的仓库。
- 只处理**用户明确给出的链接**。举例性质的链接不算，拿不准就问。(见 STYLE §2.2-5)
- **查重(必做)**：把每个待收链接在 `index.html` 里搜一遍(`grep "<url>" index.html` 或用编辑器查找)。
  **命中就停下问用户**(这篇上期已收，是要跳过、改写旧的、还是仍要新增？)——不要默默产出重复内容。
  这是两次盲测都揪出的头号缺口，务必做。

### 2. 抓取
对每个链接跑：`python skill/scripts/fetch_article.py "<url>" --out tmp_article.txt`
- 脚本已处理微信"环境异常"、图片型推文取描述、UTF-8 输出等坑。
- `tmp_article.txt` 是临时抓取产物，已在 `.gitignore` 里；**提交前确认只改了 index.html，别把 tmp 文件 add 进去**。
- 若脚本非零退出(抓不到正文)：**明确请用户粘贴该文正文，不要编造内容**。

### 3. 分类 + 写摘要(核心，照 STYLE.md)
- 先判断每篇是**干货/观点**(→精读卡片)还是**要闻/发布**(→要闻茶点一句话)。拿不准问用户。
- **聚合型早报**(一篇汇总 N 条快讯)默认压成**一条**茶点：点出头条 + 其余一句话概述，不要拆成 N 段。(STYLE §1)
- 干货：核心块用大白话点透 → 少而透的要点(每条带"比如") → 延伸块收次要信息。黑话就地翻译。
- 保留署名 + 原文链接。用**自己的话**讲，不复读作者术语。

### 4. 更新 index.html + 校验
- 决定是**并入当期**还是**新开一期**：优先问用户。**问不到用户时的兜底**——单条要闻默认并入最新期的茶点区；
  一批干货是否值得单独成期拿不准时，默认**并入当期或停下问**，不要默认新开一期(避免为少量内容硬造新 VOL)。
- 照 `references/TEMPLATE.md` 的片段改 HTML(加目录行 + 文章卡片 / 或新 tab + 新 section)。
- 跑 `python skill/scripts/validate_site.py`，**必须 0 FAIL** 再继续；若出现"重复原文链接"WARN，回到查重那步确认。
- 本地打开 `index.html` 肉眼看一眼(默认只露标题+核心、切换/展开/深浅色正常)。
  **无 GUI / headless 环境**下无法真开浏览器：以 `validate_site.py` 全过 + 结构自查代替，并在给用户的预览说明里
  注明"未做人工目检，请你打开确认"。

### 5. 预览 → 确认 → 发布
- 给用户预览(本地打开或描述改动)。**未经用户确认，绝不 push。**(发布是对外动作)
- 确认后：`git add index.html && git commit -m "<说明>" && git push origin main`。
- 告知用户线上地址(可从 `check_env.py` 的输出得到，形如 `https://<user>.github.io/<repo>/`)，约 1 分钟生效。

## 铁律(容易翻车的地方)
- **看懂 > 精炼 > 好看**；精炼靠"用大白话重讲"，不是删光例子只留结论。(STYLE §0)
- **加之前先查重**：待收链接若已在 index.html 里，停下问用户，别产出重复内容。
- **只收用户明确给的链接**；举例链接不自动收录。
- **要闻和干货分开呈现**，要闻进茶点区并自报身份，不做成无根之水；聚合早报压成一条。
- **发布前必过 `validate_site.py`；未确认不 push；改往期已发内容需用户明确要求。**
- 抓不到就请用户贴正文，**不硬编、不编造**。
- 不自动定时、不主动找文章——只在用户要求时干活。
